# No automatic fallback when the selected Input or Output Device disconnects

If the tech's chosen Input Device disconnects mid-show (cable pull, USB glitch), we decided the pipeline stops and the UI shows an explicit disconnected state, rather than automatically falling back to the system default or another available device. The same treatment applies to the Signal Generator's Output Device.

We considered auto-fallback for uninterrupted operation, but rejected it: silently switching devices would let the app keep running — appearing to work — while actually reading from (or playing to) a different device than the tech thinks. For Input, that's a measurement mic swapped for a laptop's built-in mic without the tech noticing. For Output, it's arguably worse: a test tone could suddenly start playing out of the Mac's speakers instead of the PA, audible to an audience, instead of going where intended. For a tool whose entire value is "trust what's on screen (and what's coming out of the speakers) enough to act on it during a show," a misleading-but-alive state is worse than an honest, obvious stop.
