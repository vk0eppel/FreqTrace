# Appearance Mode is manual (Dark/high-contrast Light), not tied to system appearance

FreqTrace is used in two lighting extremes that don't correlate with a user's general macOS preference: dim/dark venues (where a dark, saturated-color display is most readable) and direct sunlight/bright outdoor stages (where dark UIs wash out and glare, and a bright high-contrast display reads better). We decided to expose Appearance Mode as an explicit, manually-toggled control rather than following the system's light/dark appearance setting, and rather than trying to auto-detect ambient brightness (Macs don't expose a usable ambient light sensor for this).

This is a deliberate deviation from typical macOS conventions, where apps are expected to follow the system appearance. Dark is the default. A future auto-detection mechanism isn't ruled out, but isn't planned for v1.

**Addendum (selector moved to the View menu).** The selector originally lived in the Controls row's second line. It moved to the View menu (Dark ⌘⇧D / Light ⌘⇧L, checkmarked): the "everything on one screen" rule exists for controls and readouts a tech needs mid-show, and appearance is a set-once-per-venue setting, not a mid-show control — demoting it frees Controls-row space without violating that rule's intent. Following the system appearance was reconsidered and rejected again at that point, for the original reason: system appearance tracks the user's global preference or schedule, not the lighting at the FOH position, and FreqTrace's Light mode is a purpose-built high-contrast sunlight design, not a standard macOS light theme. Because a menu setting is less visible than an always-on-screen toggle, the choice is now persisted across launches (`AppearanceSettings`).
