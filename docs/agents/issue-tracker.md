# Issue tracker

Issues live in this repo's GitHub Issues: [vk0eppel/FreqTrace](https://github.com/vk0eppel/FreqTrace). Skills use the `gh` CLI (`gh issue create`, `gh issue list`, etc.).

PRs are **not** treated as a request surface — `/triage` only processes issues, not external pull requests (solo project, no external contributors yet).

## Future ideas

Not-yet-scheduled feature ideas live as issues with the `future` label — same tracker as active work, so they're triageable/agent-ready when it's time to build. Capture short (title + a sentence on *what* and *why*); flesh out to a full spec when scheduled. List them with `gh issue list --label future`.
